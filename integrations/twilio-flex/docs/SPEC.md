# Slates Specification for Twilio Flex

## Overview

Twilio Flex is a programmable cloud-based contact center (CCaaS) platform that enables businesses to manage customer interactions across multiple channels including voice, SMS, WhatsApp, Facebook Messenger, and web chat. Flex is a digital engagement center for sales and customer support teams that gives companies control over how they communicate with customers and prospects across all channels, and at every stage of the customer journey. It provides customizable routing, agent interfaces, analytics, and workflow automation built on top of Twilio's communication APIs.

## Authentication

The Flex API uses the same HTTP Basic Authentication as other Twilio APIs. Use your Twilio account SID as your username and your Auth Token as your password.

There are two credential options for HTTP Basic Auth:

1. **Account SID + Auth Token**: For local testing, you can use your Account SID as the username and your Auth Token as the password. You can find your Account SID and Auth Token in the Twilio Console.

2. **API Keys**: To authenticate requests to the Twilio APIs, Twilio supports HTTP Basic authentication. Use your API key as the username and your API key secret as the password. You can create an API key either in the Twilio Console or using the API. Twilio recommends using API keys for authentication in production apps.

The base URL for the Flex API is `https://flex-api.twilio.com/v1/`. Some Flex features rely on underlying Twilio products (TaskRouter, Conversations) which have their own base URLs but use the same authentication method.

Additionally, to provision a user via the Flex SDK, replace `{flex_instance_sid}` in the command with your Flex Instance SID (GOxxxxxxxxxxxxxxxxxxxxx). You'll find your Flex Instance SID in Console on the Flex Overview page.

## Features

### Interactions & Conversations Management

Create and manage customer interactions across channels (SMS, WhatsApp, web chat, voice). With Twilio Conversations and the Interactions API, you can create inbound and outbound conversations with your customers. Your agents will have more granular control on the lifecycle of a conversation and can proactively reach out to your customers. You can add/remove participants, transfer conversations between agents, and close channels.

- Supports both customer-initiated (inbound) and business-initiated (outbound) flows.
- When you create an interaction, you specify the Twilio Conversation to bind to and the TaskRouter workspace to use for routing. You can optionally declare additional routing attributes according to your interaction type.

### Task Routing (TaskRouter)

TaskRouter is an attributes-based routing system for your Flex contact center. TaskRouter uses Tasks to represent a unit of work that needs to be completed across your configured communication channels and provides a set of tools to allow you to intelligently route a task to the agent best suited to handle it.

- Manage Workspaces, Workers, Task Queues, and Workflows.
- Workers are entities that process tasks, like agents in a call center. Activities describe what Workers are doing and whether they are ready to accept a new task assignment.
- Define custom routing rules based on task and worker attributes.

### Flex Configuration

The Flex Configuration REST API resource contains a collection of properties that control the appearance and functionality of your Flex instance. You can read and update configuration properties programmatically, including UI appearance, enabled channels, and custom attributes.

- Some fields are read-only.

### Messaging Channels

Twilio supports Web chat, SMS, Facebook Messenger, and WhatsApp. You can configure message handlers (Flex Flows) that define how incoming messages on each channel are routed into the contact center.

- A Flex Flow is the logic linking a Messaging Channel, like SMS, to Flex. A Flex Flow also describes the way Flex should handle messages on this channel.

### Insights & Analytics

Flex Insights receives all of the events that are generated by your TaskRouter Workspace. It provides reporting and analytics on contact center performance including conversation data, agent activity, queue metrics, and customer information.

- Custom Metric and KPI Definition: Define your own custom metrics and KPIs. Custom Report Builder: Create custom reports and dashboards. Email-Based Alerting on Metrics: Define custom alerting thresholds. Pre-Built Reports and Dashboards: Analyze contact center performance through powerful built-in reports.
- Data can be exported via the Flex Insights API.

### Studio Flows

Twilio Studio provides a visual drag-and-drop builder for creating communication workflows (IVR, chatbot logic, routing). With the Studio REST API you can trigger Flows programmatically for outbound use cases, manage Flow definitions for automated deployments, and retrieve information about Executions for reporting.

### Plugins

Developers can build and deploy custom plugins using Twilio's Plugin Builder, allowing for the addition of new features or the modification of existing functionality without disrupting the core service.

## Events

Twilio Flex supports events through multiple mechanisms:

### Interactions Webhooks

Certain Interactions API operations trigger events related to an Interaction, Participant, or Channel. To get notified of events and state changes during Conversations, you can configure a webhook URL to receive Interaction events and then subscribe to selected events.

- There are two types of Interactions webhooks: global — get notifications for all events you subscribe to using one webhook URL for your Flex instance. Or interaction-specific — receive events only for a specific interaction.
- Available events:
  - Sent when an Interaction is created.
  - Sent when a Channel is created.
  - Sent if a Channel can't be created.
  - Sent when the Channel status is updated. The status can be active or closed.
  - Sent when a Participant is added to the Channel.
  - Sent when a Participant leaves the Channel.
- Configured via Console or via the `/v1/Instances/{instanceSid}/InteractionWebhooks` API endpoint.

### Twilio Event Streams

The Event Streams REST API provides access to a unified stream of every interaction sent or received on Twilio. Event Streams supports streaming events to multiple sinks (destinations). This is the broader Twilio eventing platform that covers Flex-relevant products like TaskRouter, Conversations, Voice, Messaging, and Studio.

- You can configure Event Streams to send events to Amazon Kinesis, Segment, or webhooks.
- You create Sink resources (destinations) and Subscription resources (event type selections) to control which events you receive.
- TaskRouter events include task created/assigned/completed/canceled, worker activity changes, reservation updates, and queue changes.
- Conversations events include conversation created/updated/removed, message added/updated/removed, and participant added/removed.
- Studio events include execution started/ended and step ended.

### TaskRouter Event Callbacks

As TaskRouter works, it generates a stream of valuable events and statistics about agent activity, task volume, and more. TaskRouter workspaces can be configured with an Event Callback URL to receive HTTP callbacks for workspace state changes (task events, reservation events, worker activity changes).

### Agent Copilot Webhooks

The webhook will trigger when either of the following events occurs: a wrap-up note is created or when an agent edits/updates a note. This is a public beta feature.
