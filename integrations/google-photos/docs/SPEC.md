# Slates Specification for Google Photos

## Overview

Google Photos is Google's photo and video storage and management service. The Google Photos APIs consist of two APIs: the **Picker API**, which lets users securely select photos and videos from their library to share with applications, and the **Library API**, which allows applications to upload, manage, and organize photos and albums that were created by the app within users' Google Photos libraries.

## Authentication

Google Photos APIs use OAuth 2.0 to handle authorization. Service accounts are not supported. No other authorization protocols are supported.

**OAuth 2.0 Flow:**

1. Register your application using the Google API Console. Google provides a client ID and a client secret.
2. Activate the Google Photos APIs in the Google API Console.
3. When your application needs access to user data, it asks Google for a particular scope of access. Google displays a consent screen to the user. If the user approves, Google gives your application a short-lived access token.
4. Some application types include additional steps, such as using refresh tokens to acquire new access tokens.

**OAuth 2.0 Endpoints:**

- Authorization endpoint: `https://accounts.google.com/o/oauth2/v2/auth`
- Token endpoint: `https://oauth2.googleapis.com/token`

**Credentials Required:**

- Client ID
- Client Secret

**Picker API Scopes:**

| Scope                                                              | Description                                                                              |
| ------------------------------------------------------------------ | ---------------------------------------------------------------------------------------- |
| `https://www.googleapis.com/auth/photospicker.mediaitems.readonly` | Access to create, get, and delete picker sessions, and to list media items for sessions. |

**Library API Scopes:**

| Scope                                                                   | Description                                                                                                                                                      |
| ----------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `https://www.googleapis.com/auth/photoslibrary.appendonly`              | Upload media items and create albums. Only allows new media to be created in the user's library and in albums created by the app.                                |
| `https://www.googleapis.com/auth/photoslibrary.readonly.appcreateddata` | Read access to media items and albums created by the app. Intended to be used together with `photoslibrary.appendonly`.                                          |
| `https://www.googleapis.com/auth/photoslibrary.edit.appcreateddata`     | Edit access to albums and media items created by the app, including organizing items in albums, changing album titles/cover photos, and media item descriptions. |

**Important Notes:**

- Applications must pass the OAuth verification review. An "unverified app" message appears until verification is complete.
- The `photoslibrary.readonly`, `photoslibrary.sharing`, and `photoslibrary` scopes were removed on April 1, 2025. The Library API can now only access content created by the app itself.

## Features

### Photo Picking (Picker API)

The Picker API lets users securely select photos and videos from their Google Photos library and share them with your application. The flow involves creating a session via the API, which returns a unique `pickerUri`. Users are directed to this URI, which opens the Google Photos app where they browse and select content.

- Users can search for photos using keywords, dates, locations, and album titles.
- For web-based applications, appending `/autoclose` to the URI automatically closes the Google Photos window after selection.
- The `pickerUri` cannot be opened in an iframe due to security reasons.
- Base URLs for accessing media content remain active for 60 minutes.

### Uploading Media (Library API)

Upload and store photos and videos generated by your app directly into users' Google Photos libraries.

- The maximum upload file size for a photo is 200 MB.
- A MIME type must be specified during the upload stage.
- Filenames are specified when creating a media item, not during the byte upload.

### Album Management (Library API)

The Library API lets you create and manage albums created by your app.

- Create albums and add media items to them.
- Change album titles and cover photos.
- Add and remove media items from albums.
- Add enrichments (text, location, map markers) to albums to provide context between media items.

### Media Item Management (Library API)

You can list, search, and retrieve albums and media items that were created by your app.

- Change descriptions for media items your app has created.
- Search within app-created content with filters including date ranges, content categories (e.g., landscapes, selfies, animals), and favorites.
- Search results can be sorted newest first or oldest first.
- Access media metadata including camera information, creation time, dimensions, and exposure details.
- The Library API no longer allows access to elements in a user's library that were not uploaded by your app.

## Events

The provider does not support events. Google Photos APIs do not offer webhooks, push notifications, or any built-in event subscription mechanism for monitoring changes to a user's library or albums.
