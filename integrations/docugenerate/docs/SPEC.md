# Slates Specification for Docugenerate

## Overview

DocuGenerate is a document generation platform that creates documents from reusable Word templates (DOCX, DOC, ODT, TXT) by merging them with dynamic data provided as JSON or spreadsheet files. It produces output in multiple formats including PDF, DOCX, DOC, ODT, TXT, HTML, and PNG, and is commonly used for generating invoices, contracts, certificates, letters, and similar business documents.

## Authentication

DocuGenerate provides secure access to its API through the use of API keys. You can get your API key from the Settings page in the DocuGenerate web app.

To authenticate, the API key needs to be included in all API requests to the server in a header:

```
Authorization: <API Key>
```

The API key is a simple string (e.g., `491c000c5fad32ed7787005b0723ad55`). There are no OAuth flows, scopes, or additional credentials required. The base API URL is `https://api.docugenerate.com/v1/`.

## Features

### Template Management

Templates are the building blocks of DocuGenerate, providing a way to generate high-quality documents with consistent formatting. With templates, you can define the structure of your documents once and then use them repeatedly to generate new documents as needed.

- You can create a template by uploading a Microsoft Word (DOCX or DOC), OpenDocument (ODT) or Plain Text (TXT) file.
- The merge tags are detected automatically. Click on them to be highlighted in the template. Unclosed tags will appear as invalid and need to be fixed.
- You can change the tag delimiters from `[Name]` to a custom one, like `«Name»` or `{Name}`.
- Templates can be listed, retrieved, updated (including re-uploading the file), and deleted.

### Enhanced Syntax

The enhanced syntax offers additional power and flexibility in designing your templates, allowing for complex expressions, advanced formatting, and more.

- Can be enabled or disabled per template.
- When using enhanced syntax, merge tag names cannot contain whitespace characters (use underscores instead).

### Document Generation

The data parameter should be a JSON array with values for merging and creating documents from the template. For each object in the array, a document will be generated by replacing the merge tags with the object's values.

- Output format can be .pdf, .docx, .doc, .odt, .txt, .html, or .png.
- Data can be provided as JSON or uploaded as a spreadsheet file (XLSX, XLS, ODS, CSV, or TSV).
- Configurable options include: output name, output format, output quality, page break between items, and whether to combine into a single file (`single_file`).
- It's also possible to receive generated documents in binary format, bypassing storage entirely.
- Generated documents can be listed (by template), retrieved, updated (name), and deleted.

### PDF Attachments

With DocuGenerate's API, it is possible to add an attachment when calling the Generate Document endpoint by using the `attach` parameter. The file can be specified as a URL or a Base64-encoded data URI. This parameter is only applicable if output_format is .pdf (or a PDF/A version).

- The attach parameter also supports an array of files, allowing you to attach multiple files to a single PDF.

### PDF Merging

PDF merging allows you to append additional PDF files at the end of your generated documents, making it easy to include supplementary information, terms and conditions, appendices, or any other relevant content. This feature is particularly useful for creating documents that combine dynamically generated content with static PDF materials.

- Controlled via the `merge_with` parameter, accepting URLs or Base64-encoded data URIs.
- Supports an array of PDF files, allowing you to merge multiple PDFs at the end of your generated document.
- Only applicable when output format is PDF or PDF/A.

### Template Versioning

Once versioning is enabled for a template, a new version is automatically saved whenever you update the template file. Changes to template metadata (like the name or other settings) do not create new versions. Only actual changes to the template file itself trigger the creation of a new version.

- Template version management is only available through the web app. When using the API, you'll always interact with the current version of a template. You cannot list, restore, or delete template versions via the API.
- You can enable or disable versioning for a template via the API, but you cannot manage the versions themselves.

### Multi-Region Processing

With DocuGenerate's multi-region processing feature, you can use regional API endpoints to ensure that your templates and documents are processed in the region of your choice. This feature enables you to meet local data regulations (such as GDPR compliance), reduce latency for global users, and optimize performance.

- Regional endpoints are available (e.g., `https://api.eu.docugenerate.com/` for EU, `https://api.au.docugenerate.com/` for Australia).
- You can choose a default region for your account, and also configure regions at the template level.

## Events

The provider does not support webhooks or event subscriptions through its API. The "Watch New Document" and "Watch New Template" triggers available in third-party integration platforms like Make are polling-based triggers that activate whenever a new document is generated or a new template is created, rather than native webhook mechanisms provided by DocuGenerate's API itself.
