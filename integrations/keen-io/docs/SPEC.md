# Slates Specification for Keen.io

## Overview

Keen.io is an event streaming and analytics platform that allows developers to collect, store, and analyze custom event data via API. It is built on Apache Kafka®, Storm®, and Cassandra®. It provides a Stream API for tracking any event (visits, clicks, signups, purchases, etc.), a Compute API for running analyses, an Access API for security, and visualization tools for building dashboards.

## Authentication

Keen.io uses API key-based authentication. Keen supports two mechanisms for authenticating HTTP API requests, all requiring API Keys for the project you want to use.

All requests require a **Project ID** to identify which project you are working with. API keys are then passed either via:

1. **Authorization header**: Set the `Authorization` HTTP header to the API key value. This method works for every API call.
2. **Query parameter**: You can also authenticate requests with a query string parameter called `api_key`.

### API Key Types

Keen.io provides four types of API keys, each found on the project's "Access" tab at `https://keen.io/project/PROJECT_ID`:

- **Master Key**: The most powerful API key. It can be used to authenticate for any API call. It is required to perform administrative functions, such as deleting data. Should never be exposed publicly.
- **Write Key**: An API key specifically for writing data. Any API request that writes data to Keen can use this.
- **Read Key**: An API key for querying and extracting data. Used to run analyses and extract data from Keen.
- **Access Keys**: These are custom keys that you define to give access to subsets of data and operations on that data. Permissions include "writes", "queries", "saved_queries", "cached_queries", "datasets", and "schema". Access Keys can also include filters and autofill properties to scope data access per customer.

### Base URL

All API requests are made to `https://api.keen.io/3.0/projects/PROJECT_ID/`.

## Features

### Event Streaming (Data Collection)

Record custom events into event collections via the API. Events are arbitrary JSON objects with custom properties. Event collections need not exist in advance; Keen.io will create them on the first request. Supports single event recording, batch uploads of multiple events to multiple collections, and ingestion via image beacons and redirects for tracking in environments where JavaScript is unavailable.

### Data Analysis (Compute API)

Run a wide range of analytical queries on your event data. The API supports aggregations including minimum, maximum, average, sum, and standard deviation for numeric values. Also supports count, count unique, select unique, percentile, median, and extraction queries. Funnel analysis includes advanced features like inverse steps ("did not do"), optional steps, and returning actor IDs for any step in the sequence.

Key query parameters include:

- **Timeframe**: Specify absolute or relative periods of time over which to run analyses.
- **Interval**: Return results within a given interval of time, such as minutely, daily, or monthly.
- **Groups**: Sub-divide analysis results categorically, based on values for a given property.
- **Filters**: Refine the scope of events to be included in an analysis, based on property values.

### Saved and Cached Queries

Saved queries are a way to revisit favorite metrics. Rather than entering the same query parameters over and over again, queries can be easily saved, edited, and shared with teammates. When you create a Cached Query, you can control the "refresh_rate", which defines the interval at which the query is automatically computed. Cached queries are automatically configured with a refresh rate of every 4 hours, configurable up to 48 hours.

### Cached Datasets

Pre-computed, indexed query results that can be retrieved with sub-second response times. Cached Queries and Cached Datasets lookups do not contribute to ad-hoc query rate limiting, making them ideal for building customer-facing embedded analytics. Datasets are created with a query definition, an interval, and an index property.

### Data Enrichment (Add-Ons)

A data enrichment is a powerful add-on to enrich the data you're already streaming to Keen. Available enrichments include:

- **IP to Geo Parser**: Parse an IP address into geographic information. **User Agent Parser**: Parse a user agent string into device, browser, and OS information. **URL Parser**: Parse a well-formed URL into queryable pieces. **Referrer Parser**: Parse a well-formed referrer URL into its source information.
- **Datetime Parser**: Parse timestamps into components like day of the week, hour, etc.

Enrichments are configured by including add-on directives within the `keen` namespace of event payloads.

### Access Key Management

An Access Key is an API Key generated by the API to identify the source or user making a request. You can programmatically generate, revoke, or modify Access Keys. For example, if you wanted customer-facing analytics, Access Keys allow individual customers to see their own data. Access Keys can also restrict where a user can send data or automatically include other data properties.

### Data Extraction and Deletion

Extract full-resolution event data from collections via the API or CSV download. The API allows you to delete events from event collections, optionally supplying filters, timeframe, or timezone to narrow the scope. Deletion requires the Master Key.

### Amazon S3 Streaming

When you activate this integration, Keen will stream your full-resolution, enriched event data to Amazon S3 in your Amazon account. Data is streamed to S3 in fixed time increments (default is every 5 minutes).

### Schema Inspection

Inspect schema information for a single event collection or all event collections in a project, including property names and types.

## Events

The provider does not support outbound webhooks or event subscriptions. Keen.io is designed to _receive_ event data (via its Stream API or inbound webhooks), not to push notifications when something changes. It offers an API URL that can be used as an endpoint for any service that produces webhooks for inbound data collection, but there is no mechanism for subscribing to changes or events within Keen.io itself.
