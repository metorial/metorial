# Slates Specification for Wachete

## Overview

Wachete is a web content monitoring service that tracks changes on web pages and notifies users when content is updated. It supports monitoring single pages, entire portals via crawling, password-protected pages, and documents (PDF, DOC). Users can configure alerts, notification channels, and monitoring frequency.

## Authentication

Wachete uses API key-based authentication with a token exchange mechanism.

1. Obtain your `userId` and `apiKey` from your Wachete profile settings.
2. Exchange these credentials for a bearer token by calling:

```
POST https://api.wachete.com/thirdparty/v1/user/apilogin
Content-type: application/json

{
  "userId": "your-user-id",
  "apiKey": "your-api-key"
}
```

3. The response contains a `token` field. Use this token as a Bearer token in the `Authorization` header for all subsequent API requests:

```
Authorization: bearer <token>
```

There are no OAuth flows or scopes. The token grants access to all resources associated with the authenticated account.

## Features

### Monitor Management (Wachets)

Create, update, retrieve, and delete monitors ("wachets") for tracking web page content changes. Each wachet is configured with:

- **URL and XPath**: The target page and the specific part of the page to monitor.
- **Job Type**: `SinglePage` for a single URL, or `Portal` for automatic crawling of subpages.
- **Recurrence**: How often the page is checked, specified in seconds.
- **Alerts**: Conditions that trigger notifications, such as `Error` (page errors) or `NotEq` (content changed).
- **Notification Endpoints**: Where to send alerts — supports `Webhook` and `Email` types.
- **Dynamic Content**: Option to enable JavaScript rendering for pages with dynamically loaded content.
- **Proxies**: Monitor from specific geographic locations (e.g., `us`, `gb`).
- **Portal Crawling Options**: For portal-type wachets, configure crawling depth (1–3) and URL filters using simple string matching or regex for include/exclude rules.

### Notification Retrieval

Retrieve a list of change notifications generated by monitors. Notifications can be filtered by a specific wachet ID and by date range (ISO format `from`/`to` parameters).

### Historical Data Retrieval

Access the collected content values (history) for a specific wachet over time. Options include:

- Returning raw content values or diff output (Google diff format).
- Filtering by date range.
- Supports continuation tokens for retrieving large datasets.

### Folder Organization

List and browse the folder structure used to organize wachets. Folders can be navigated by specifying a `parentId`; omitting it returns root-level contents.

## Events

Wachete supports **webhooks** as a notification endpoint for monitors. When creating or updating a wachet, you can configure a webhook URL as a notification endpoint. Wachete will send an HTTP request to that URL when a configured alert condition is triggered.

### Content Change Events

- **Description**: Triggered when monitored web page content changes or an error occurs, based on the alert conditions configured on the wachet (e.g., `NotEq` for content differences, `Error` for page errors).
- **Configuration**: Set the `notificationEndpoints` array on a wachet to include `{"type": "Webhook", "value": "https://your-endpoint-url"}`. Alert types (`NotEq`, `Error`) determine which conditions trigger the webhook.
