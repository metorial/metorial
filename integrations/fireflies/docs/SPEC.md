# Slates Specification for Fireflies

## Overview

Fireflies.ai is an AI-powered meeting assistant that automatically records, transcribes, and analyzes voice conversations from platforms like Zoom, Google Meet, and Webex. Fireflies is an AI voice assistant that helps transcribe, take notes, and complete actions during meetings. The API provides programmatic access to meeting transcripts, AI-generated summaries, user management, and audio upload capabilities via a GraphQL interface.

## Authentication

Fireflies uses **API Key** authentication.

- To query any endpoint, you must pass your API Key in the Authorization header for every Request query.
- The API key is passed as a Bearer token: `Authorization: Bearer <your_api_key>`
- Log in to your Fireflies.ai account and navigate to the API section to generate your unique API key for authentication. Specifically, go to the Integrations section in the left sidebar and click "Fireflies API" to copy the key.
- The public API endpoint is `https://api.fireflies.ai/graphql`.
- All requests are POST requests to this single GraphQL endpoint with `Content-Type: application/json`.
- Usage beyond 50 API calls per day requires a Business account.

**Super Admin**: Enterprise tier users with the Super Admin role have elevated access, allowing team-wide operations such as setting up webhooks for all meetings owned by the team.

## Features

### Transcript Management

Retrieve, search, and manage meeting transcripts. You can list all transcripts with optional filters (keyword search, date range, participants), fetch full transcript details including sentences with timestamps and speaker labels, and delete transcripts. You can update the title of a meeting transcript and update privacy settings with options like `link`, `owner`, `participants`, `teammatesandparticipants`, or `teammates`.

### Audio Upload & Transcription

The uploadAudio mutation allows you to upload audio files to Fireflies.ai for transcription. You provide a URL to the audio file along with a title, and optionally specify attendees (with display name, email, phone number), a language code, a webhook URL for that specific upload, and a custom `clientReferenceId` to track the upload. Free plan users cannot upload audio files.

### AI-Generated Meeting Intelligence

Transcripts include AI-generated summaries, action items, keywords, sentiment analysis, and meeting analytics. You can access AI-driven analytics such as sentiment, topics, and engagement data for a specific meeting transcript.

### AskFred AI Assistant

AskFred is Fireflies' AI assistant that can answer questions about your meetings using natural language. You can create conversation threads tied to a specific transcript, ask follow-up questions within a thread, list threads, retrieve thread details, and delete threads. Use AskFred queries and threads to ask questions and receive AI responses based on meeting content.

### User & Team Management

Get all relevant information about a user in your team, including roles, email address, most recent meeting, and the number of transcription minutes consumed by the user. You can query individual users or all team members, and set user roles.

### Live Meeting Controls

Add the Fireflies bot to a live meeting for automatic recording and transcription. You can pause and resume meeting recordings during sensitive discussions, and create live action items or soundbites during active meetings.

### Soundbites (Bites)

Create soundbites from key moments and share them with teammates or in a channel. Soundbites are short clips extracted from meeting recordings that highlight important segments.

### Channels

Manage Fireflies channels within a workspace — list all channels and assign meetings to channels. Channels can be public or private and serve as organizational containers for meetings.

### Contacts

Query contact information associated with meetings.

### AI Apps

Retrieve results generated by AI applications installed in your Fireflies workspace.

## Events

Fireflies supports webhooks for event-based notifications.

### Transcription Completed

Triggers when a meeting has been processed and the transcript is ready for viewing. The webhook payload includes a `meetingId` identifier for the meeting/transcript (MeetingId and TranscriptId are used interchangeably). The payload also includes an `eventType` field and an optional `clientReferenceId` if one was set during upload.

- **Global webhook**: Configure a webhook URL in the Fireflies dashboard (Developer Settings tab) that sends notifications for all subscribed events. This webhook will only be fired for meetings that you own. Enterprise tier with Super Admin role supports team-wide webhooks.
- **Per-upload webhook**: You can include a webhook URL as part of an upload audio request, which will only send notifications for that singular audio upload request.
- **Authentication**: Webhook requests include an `x-hub-signature` header containing a SHA-256 HMAC signature of the payload. You can configure a shared secret (16-32 characters) in the Developer Settings to verify authenticity.
