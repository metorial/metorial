# Slates Specification for Starton

## Overview

Starton is a Web3 infrastructure API that allows developers to deploy and interact with smart contracts, manage blockchain transactions, store files on IPFS, and monitor on-chain events across EVM-compatible blockchains. It provides a single API and frontend to standardize the deployment and interaction of smart contracts, supporting networks such as Ethereum, Polygon, BNB Smart Chain, Avalanche, Arbitrum, and Polygon zkEVM.

## Authentication

Starton uses **API key** authentication. To use Starton from your code, you will need to authenticate yourself to the API through an API key.

- **Header**: The API key is passed as the `x-api-key` header in all HTTP requests.
- **Base URL**: `https://api.starton.com/v3`
- **Key management**: The API keys generated by Starton are linked to a specific project, which means you could have multiple API keys. API keys are only accessible once at creation. On the Developer section, you can generate new API keys or revoke the previous ones.
- **Scopes**: When creating an API key, you select the scope of your API key. Scopes can be configured per service (e.g., Deploy, Interact, Wallet) with granular permissions such as Create, Edit, Sign, and Depot.
- **Test vs. Production keys**: You can select "Test API key" for development purposes or create production keys.

Example header:

```
x-api-key: YOUR_API_KEY
```

## Features

### Smart Contract Deployment

Deploy smart contracts on EVM-compatible networks using your own contracts by importing bytecode, or use a collection of customizable contract templates including ERC20, ERC721, ERC1155, and more. You can also import an already deployed smart contract.

- Templates are pre-audited and ready to use from Starton's library.
- You must specify a network, signer wallet, and contract parameters.

### Smart Contract Interaction

The API allows you to interact with every smart contract on EVM-based blockchains easily. You can call read and write functions on any deployed or imported smart contract.

- Supports calling any function defined in the contract's ABI.
- The contract must be deployed or imported into your Starton project first.

### Transaction Management (Relayer)

The Relayer is an API to create transactions, deploy or interact with smart contracts. Starton handles transactions and providers, deals with nonce issues, keys management, and monitors blockchains.

- You can decide on the appropriate gas fee strategy at the transaction or project level.
- The Relayer assigns nonces with precision and replaces stuck transactions, preventing transaction losses.
- Includes a load balancer to leverage multiple RPC node providers, and a queue transaction system to ensure no transactions are lost.

### IPFS Storage

With Starton IPFS you can store all your assets on a distributed network powered and secured by IPFS. By storing your assets on IPFS through the API, you can ensure their immutability and accessibility at all times.

- Upload files (images, metadata JSON, etc.) and receive a CID (content identifier).
- Commonly used for NFT metadata and assets.

### Blockchain Monitoring (Watchers)

Monitor inspects every block on all supported blockchains to continuously check for activity. Watchers represent conditions that are checked upon inspection of the block.

- Create watchers to track address activity, token transfers, mints, approvals, and custom contract events.
- Configure the number of confirmation blocks before triggering.
- Notifications are delivered via webhooks.

### Wallet Management (KMS)

Starton Wallet uses a KMS (Key Management System). You can create wallets on Starton's built-in KMS or import your own KMS (e.g., AWS KMS) for production use.

- Wallets created on Starton KMS are used to sign transactions server-side.
- Granting full access to Starton enables you to dynamically create new wallets with the Starton API.

### Token Data

Get token balances and transfers and deploy your own tokens through one single API.

## Events

Starton supports event monitoring through its **Watcher** system, which delivers notifications via **webhooks**. All events triggered by Starton are sent to webhooks. Starton provides a signature within the request's headers (`starton-signature`) so you can verify webhook authenticity.

When creating a watcher, you specify an address, a blockchain network, an event type, a webhook URL, and the number of confirmation blocks to wait before triggering.

### Address Activity Events

- **ADDRESS_ACTIVITY**: Monitors and notifies of any activity related to the provided address. Covers both incoming and outgoing transactions.
- **ADDRESS_RECEIVED_NATIVE_CURRENCY**: Triggers when an address receives native currency (e.g., ETH, MATIC).
- **ADDRESS_SENT_NATIVE_CURRENCY**: Triggers when an address sends native currency.

### ERC20 Token Events

- **EVENT_TRANSFER**: Triggers when an ERC20 contract emits a transfer event, to track when tokens are moved.
- **EVENT_MINT**: Triggers when an ERC20 contract emits a transfer event where the sending address is 0x0, to track when new tokens are created.
- **EVENT_APPROVAL**: Triggers when an ERC20 contract emits an approval event, to track when a new allowance has been granted or revoked.

### ERC721 (NFT) Events

- **ERC721_EVENT_TRANSFER**: Triggers when an ERC721 contract emits a transfer event, to track when an NFT is moved.

### ERC1155 (Multi-Token) Events

- **ERC1155_EVENT_TRANSFER_BATCH**: Triggers when an ERC1155 contract emits a batch transfer event, to track when several NFTs are moved.
- **ERC1155_EVENT_TRANSFER_SINGLE**: Triggers when a single ERC1155 token is transferred.

### Custom Events

- **EVENT_CUSTOM**: You can specify the event you want to monitor from the ABI of your contract, allowing monitoring of any custom smart contract event.
