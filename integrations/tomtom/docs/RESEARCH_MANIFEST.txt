TRAFFIC FLOW API
================================================================================

Status: ACTIVE
Last Updated: Every minute
Documentation: COMPLETE

Endpoints Researched:
  1. Flow Segment Data
     - URL: /traffic/services/4/flowSegmentData/{style}/{zoom}/{format}
     - HTTP: GET
     - Parameters: 10+ (point, style, zoom, format, unit, thickness, openLr, etc.)
     - Response: JSON with speed, travel time, confidence, road closure data

Data Sources: 650M+ GPS-enabled devices
Coverage: Global

TRAFFIC INCIDENTS API
================================================================================

Status: ACTIVE
Last Updated: Every minute
Documentation: COMPLETE

Endpoints Researched:
  1. Incident Details (GET)
     - URL: /traffic/services/5/incidentDetails
     - HTTP: GET
     - Max IDs: 5 per request
     - Parameters: bbox OR ids, language, categoryFilter, timeValidityFilter
     
  2. Incident Details (POST)
     - URL: /traffic/services/5/incidentDetails
     - HTTP: POST
     - Max IDs: 100 per request
     - Body: JSON with array of incident IDs
     - Parameters: ids (in body), language, categoryFilter, timeValidityFilter

Response: GeoJSON FeatureCollection with incident properties
Incident Data: Location, delay time, affected roads, closure status, etc.

GEOFENCING API
================================================================================

Status: DEPRECATING (12-month deprecation period)
Documentation: COMPLETE

Services & Endpoints Researched:

  CONFIGURATION SERVICE
    1. Register (POST)
       - URL: /geofencing/1/register
       - Returns: Admin Key

  PROJECTS SERVICE
    2. List Projects (GET)
       - URL: /geofencing/1/projects
    3. Get Project Details (GET)
       - URL: /geofencing/1/projects/{projectId}
    4. Edit Projects (PUT)
       - URL: /geofencing/1/projects/{projectId}

  FENCES SERVICE
    5. List Fences (GET)
       - URL: /geofencing/1/fences
    6. Get Fence Details (GET)
       - URL: /geofencing/1/fences/{fenceId}
       - Parameters: showGeoJson (optional)
    7. Create Fence (POST)
       - URL: /geofencing/1/fences
    8. Edit Fence (PUT)
       - URL: /geofencing/1/fences/{fenceId}
    9. Delete Fence (DELETE)
       - URL: /geofencing/1/fences/{fenceId}

  OBJECTS SERVICE
   10. List Objects (GET)
       - URL: /geofencing/1/objects
   11. Get Object Details (GET)
       - URL: /geofencing/1/objects/{objectId}
   12. Create Object (POST)
       - URL: /geofencing/1/objects
   13. Edit Object (PUT)
       - URL: /geofencing/1/objects/{objectId}
   14. Delete Object (DELETE)
       - URL: /geofencing/1/objects/{objectId}

  REPORT SERVICE
   15. Report Request (GET/POST)
       - URL: /geofencing/1/report/{projectId}
       - GET/POST both supported
       - Parameters: point, object, range, timestamp (POST only)
       - Response: Inside/outside fence features with distances

Authentication: API Key + Admin Key (both required for management)
Transitions: Detects object entry/exit through fence borders
Data Retention: 3 months active, 6 months archive

LOCATION HISTORY API
================================================================================

Status: ACTIVE
Documentation: COMPLETE

Services & Endpoints Researched:

  CONFIGURATION SERVICE
    1. Register (POST)
       - URL: /locationHistory/1/register
       - Returns: Admin Key
    2. List Settings (GET)
       - URL: /locationHistory/1/settings
    3. Set Settings (PUT)
       - URL: /locationHistory/1/settings

  OBJECTS SERVICE
    4. List Objects (GET)
       - URL: /locationHistory/1/objects
    5. Get Object Details (GET)
       - URL: /locationHistory/1/objects/{objectId}
    6. Create Object (POST)
       - URL: /locationHistory/1/objects
    7. Edit Object (PUT)
       - URL: /locationHistory/1/objects/{objectId}
    8. Delete Object (DELETE)
       - URL: /locationHistory/1/objects/{objectId}

  POSITION HISTORY SERVICE
    9. Send Position (POST)
       - URL: /locationHistory/1/history/positions
       - Body: GeoJSON Feature with Point geometry
       - Fields: type, geometry, object, timestamp
   10. Get Position History (GET)
       - URL: /locationHistory/1/history/positions/{objectId}
       - Parameters: from (required), to, maxResults, pageNumber
       - Time range: Max 24 hours per request
   11. Last Position (GET)
       - URL: /locationHistory/1/history/positions/{objectId}/latest

Response Fields: timestamp, estimatedSpeed, estimatedDirection, altitude, coordinates
Authentication: API Key + Admin Key (both required)
Shares Objects: With Geofencing API (same account)
Data Retention: 3 months active, 6 months archive

================================================================================

Status: DECOMMISSIONING (January 31, 2027)
Documentation: COMPLETE

Services & Endpoints Researched:

  CONFIGURATION SERVICE
    1. List Settings (GET)
       - URL: /notifications/1/settings
    2. Set Settings (PUT)
       - URL: /notifications/1/settings

  CONTACT GROUPS SERVICE
    3. Create Contact Group (POST)
       - URL: /notifications/1/groups
       - Body: name, webhookUrls (max 20), emails (max 20)
    4. List Contact Groups (GET)
       - URL: /notifications/1/groups
       - Parameters: maxResults, pageNumber
    5. Get Group Details (GET)
       - URL: /notifications/1/groups/{groupId}
    6. Edit Contact Group (PUT)
       - URL: /notifications/1/groups/{groupId}
       - Full replacement update
    7. Partial Edit Group (PATCH)
       - URL: /notifications/1/groups/{groupId}
       - Partial update
    8. Delete Contact Group (DELETE)
       - URL: /notifications/1/groups/{groupId}

  NOTIFICATIONS HISTORY SERVICE
    9. List Notifications History (GET)
       - URL: /notifications/1/history
       - Parameters: from (required), to, maxResults, pageNumber
   10. Clear History (DELETE)
       - URL: /notifications/1/history
       - Parameter: before (ISO 8601 timestamp)

Notification Methods: Webhooks, Email
Limits: 20 webhooks + 20 emails per contact group
History Retention: 7 days (auto-deleted)
Authentication: API Key only

AUTHENTICATION DETAILS DOCUMENTED
================================================================================

API Key:
  - Query Parameter: key=YOUR_API_KEY
  - Used By: All endpoints (required)
  - Generated In: TomTom Developer Portal
  - Scope: Can be restricted to specific products
  
Admin Key:
  - Query Parameter: adminKey=YOUR_ADMIN_KEY
  - Used By: Management endpoints (Geofencing, Location History)
  - Generated Via: POST to /register endpoint
  - Required For: Create, Edit, Delete operations

Regional Endpoints:
  - Primary: https://api.tomtom.com
  - Korea: https://kr-api.tomtom.com

================================================================================
KEY PARAMETERS ACROSS ALL APIs
================================================================================

Common Query Parameters:
  - key (all endpoints) - API Key
  - adminKey (management endpoints) - Admin Key
  - from (history endpoints) - ISO 8601 timestamp
  - to (history endpoints) - ISO 8601 timestamp
  - maxResults (pagination) - Integer 1-100
  - pageNumber (pagination) - Integer ≥1
  - language (traffic API) - Language code

Common Request Body Fields:
  - type (always "Feature" for GeoJSON)
  - geometry (coordinates, type)
  - properties (custom fields)
  - name (resource names)
  - id (resource identifiers)

Common Response Fields:
  - id (resource identifier)
  - name (resource name)
  - properties (custom attributes)
  - geometry (coordinates)
  - timestamp (ISO 8601)
  - resultInfo (pagination details)

================================================================================
DATA FORMATS DOCUMENTED
================================================================================

Coordinates:
  Format: longitude,latitude[,altitude]
  Projection: WGS84 (EPSG:4326)
  Example: 4.8432,52.3745,100

Timestamps:
  Format: ISO 8601
  Pattern: YYYY-MM-DDThh:mm:ss[Z]
  Example: 2024-03-14T14:30:45Z

GeoJSON:
  Type: Feature or FeatureCollection
  Geometry Types: Point, LineString, Polygon
  Coordinate Order: [longitude, latitude, altitude]

Pagination:
  Default: 100 results per page
  Max: 100 results per page
  Parameters: maxResults, pageNumber
