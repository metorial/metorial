# Slates Specification for Deepgram

## Overview

Deepgram is a voice AI platform that provides APIs for speech-to-text (STT), text-to-speech (TTS), text intelligence, and conversational voice agents. It supports both real-time streaming and pre-recorded audio processing, with models available in 45+ languages. The platform can be used cloud-hosted or self-hosted.

## Authentication

Deepgram supports two authentication methods:

### API Key Authentication

Deepgram's API uses API keys to authenticate requests. API keys are associated with Deepgram Projects, which organize all of your Deepgram resources and consist of a set of users, a set of API keys, and billing and monitoring settings.

- API keys are passed in the `Authorization` header using the `Token` scheme: `Authorization: Token <API_KEY>`
- You must create your first API Key using the Console. All API requests must be made over HTTPS.
- When you create an API key, you assign it a Role, which determines which actions it can be used to perform in the associated Project.

**Roles and Scopes:**

API keys are assigned one of three project roles, each with increasing permissions:

- **Member**: Can read/write project settings, manage own keys, and read/write usage data.
- **Admin**: Member permissions plus managing other members, invitations, and reading billing data.
- **Owner**: Full access including project destruction, billing write, and managing all users at all levels.

### Token-Based Authentication (Temporary JWT)

Generates a temporary JSON Web Token (JWT) with a 30-second TTL (Time To Live) for Deepgram APIs. A temporary token is a secure, authentication credential that provides time-limited access to Deepgram's APIs. These tokens have a 30 second Time To Live (TTL) and are designed specifically for short-lived, secure API access scenarios, particularly in client-side applications where long-term credential storage is not recommended.

- Tokens are generated by calling `POST https://api.deepgram.com/v1/auth/grant` using an existing API key (requires Member scope or higher).
- Tokens are passed using the `Bearer` scheme: `Authorization: Bearer <JWT_TOKEN>`
- Temporary tokens are ideal for client-side/untrusted applications because they allow clients to directly access the Deepgram API. Normal API keys, to be secure, require proxying traffic through your own servers where the API key is stored.

## Features

### Speech-to-Text (Pre-recorded)

Transcribe pre-recorded audio files or audio from a URL. Supports various audio formats and 45+ languages. Key options include model selection (e.g., `nova-3`), smart formatting (auto-formats currencies, phone numbers, emails), punctuation, diarization (speaker identification), language detection, and keyword boosting. Results can include word-level timestamps and confidence scores.

### Speech-to-Text (Live Streaming)

Automatically transcribe live streaming audio in real time via WebSocket connections. Supports interim (partial) results for low-latency display, end-of-speech detection, and utterance segmentation. The `Flux` model is purpose-built for conversational STT with built-in turn detection and interruption handling for voice agent use cases.

### Text-to-Speech

Convert text into natural-sounding speech audio. Available as both a single REST request and a continuous WebSocket text stream for real-time voice synthesis. Features 40+ English voices with localized accents. Configurable options include voice/model selection (e.g., `aura-2`), encoding format, sample rate, bitrate, and audio container format.

### Voice Agent

Deepgram's Voice Agent API provides a single, unified API that integrates speech-to-text, LLM reasoning, and text-to-speech with built-in support for real-time conversational dynamics. Connected via WebSocket, it orchestrates listening, thinking, and speaking in a single session. Developers can configure the STT model, TTS model/voice, and LLM provider (Deepgram-hosted or bring-your-own, e.g., OpenAI). Supports defining callable functions (tool use), injecting conversation context/history, dynamic prompt updates, and mid-conversation model changes. Handles barge-in (interruptions) and turn-taking natively.

### Text Intelligence

Analyze text for various insights. Once you have the transcript, Deepgram's intelligence features allow you to extract value from it. This includes sentiment analysis to gauge a speaker's emotion, topic detection to categorize conversations, and summarization to create concise overviews. Also supports intent detection. Can be applied to pre-recorded transcription results or standalone text input.

### Project Management

Administer your Deepgram account: Manage projects and project members, manage project invitations, manage user scopes, get billing balances, get billing usage breakdown, get project requests, and manage API keys. You can use a single Deepgram project to serve each of your customers and even tag their respective usage. Deepgram allows you to tag your API Keys and/or individual requests. You can then rely on those tags to filter usage and bill your customers accordingly.

### Model Discovery

Query available Deepgram models and retrieve model metadata, useful for dynamically selecting the appropriate model for a given task or language.

## Events

Deepgram supports callbacks (webhooks) for asynchronous processing of both speech-to-text and text-to-speech requests. These are not event subscriptions in the traditional sense, but per-request callback URLs provided at the time of the API call.

### Speech-to-Text Callback

Speech-to-text Callback allows you to have your submitted audio processed asynchronously. Deepgram's Callback feature allows you to supply a callback URL to which transcriptions can be returned. When passed, Deepgram will immediately respond with a request_id before processing your audio asynchronously.

- Provide a `callback` URL parameter with your transcription request.
- Supports `POST` (default) or `PUT` via the `callback_method` parameter.
- Callback URLs may use `http://`, `https://`, `ws://`, or `wss://` protocols.
- There are two main methods for authenticating callback requests: using Basic Auth and utilizing the dg-token request header.
- If the HTTP status code of the response to the callback POST request is unsuccessful (not 200-299), Deepgram will retry the callback up to 10 times with a 30 second delay between attempts.

### Text-to-Speech Callback

A text-to-speech Callback allows you to have your submitted text processed asynchronously. Deepgram's Callback feature allows you to supply a callback URL to which generated text-to-speech audio can be returned. When passed, Deepgram will immediately respond with a request_id before processing your text asynchronously.

- Same configuration options as STT callbacks (`callback`, `callback_method` parameters).
- Same retry behavior and authentication mechanisms apply.
