## Overview

ClassMarker is a web-based online quiz and exam platform used by businesses and educational institutions to create, administer, and grade tests and assessments. Its API allows integration with LMS or CRM software to customize integrations and manage exam data, including retrieving results, managing questions, and controlling exam access.

## Authentication

ClassMarker uses API key and secret-based authentication with HMAC signing.

To use the ClassMarker API, you need an API key and secret. An API key and API secret can be created under the My Account / API/Webhooks section of ClassMarker when logged in.

Each request needs to include your API key, a lowercase SHA256 signature, and a timestamp. The signature is generated by computing a SHA256 hash of the API secret combined with the current Unix timestamp (in seconds).

Request format:

```
https://api.classmarker.com/v1.json?api_key={API_KEY}&signature={SHA256_SIGNATURE}&timestamp={UNIX_TIMESTAMP}
```

There is a 5-minute window of time variance allowed from the sent timestamp.

Each API key has permissions set on which groups it is allowed to request quiz results from. ClassMarker accounts can have multiple API key and API secret combinations created for separate groups/exam access.

Your API secret must be kept private and secure.

The API supports both JSON and XML response formats (specified via the URL path, e.g., `/v1.json` or `/v1.xml`).

## Features

### Exam Results Retrieval

Retrieve test results from your ClassMarker account, either for all groups/links or for specific group/exam or link/exam combinations. You specify a timestamp, and only quiz results completed after that timestamp are returned. Results include user details (name, email), scores (percentage, points scored, points available), timing information, duration, and grading status. To retrieve individual test questions, user responses, and category results, webhooks must be used instead of the API.

- The `finishedAfterTimestamp` parameter acts as an offset to avoid retrieving duplicate results.
- The `finishedAfterTimestamp` must be less than three months old.
- Optionally includes a unique `view_results_url` per result for reviewing formatted results.

### Groups, Links, and Tests Discovery

Retrieve a list of group, link, and test names and IDs your API key has permission to access. This can be used to see when extra groups or links have been added. This is useful for discovering available exams and obtaining the IDs needed for other API calls.

### Question Management

ClassMarker's API lets you read, add, and edit both categories and questions, allowing you to create and approve questions externally before adding them to your account.

- Supported question types for full CRUD include multiple choice and similar types. Free Text, Grammar, and Matching question types are not fully supported — only GET requests are available for those types.
- Questions can be assigned to categories, given point values, feedback text, and answer options.
- Questions and exams can be organized into categories, consisting of parent and sub-categories.

### Access List Management

Control who can take exams by managing access lists via the API. You can add and remove access list items (codes) using the API, for example when a new user registers for a course or when a user passes an exam and needs access to the next one.

- Each link can have one access list assigned to it.
- Up to 100 codes can be removed per request.

### Exam Link Integration

When distributing exams via links, you can pass user details (first name, last name, email, user ID, access code) as URL parameters to pre-populate exam taker information. The `cm_user_id` value is saved with exam results and can be retrieved via webhooks and API requests. You can also require that exams only be taken when a `cm_user_id` is supplied and limit each user to one attempt.

## Events

ClassMarker supports webhooks that deliver exam results in real time.

### Exam Results Completed

Webhooks allow test results to be securely delivered to your systems/website in real time. When a webhook is assigned to a test and the test is completed, the results are sent in real time to your webhook endpoint URL.

- The webhook payload includes the same core result data as the API (user info, scores, duration, etc.) plus individual test questions, user responses, and category-level results.
- The webhook must be assigned to a group or link. You can use the same webhook for all groups and links or create multiple webhooks for separate groups and links.
- Payloads are signed with an HMAC-SHA256 signature using a webhook-specific secret phrase, sent in the `X-ClassMarker-Hmac-Sha256` header for verification.
- Results may change after initial delivery if the exam requires essay grading or is manually re-graded. In both cases, results can be manually resent via the webhook.
- ClassMarker will retry a failed webhook within minutes of it first failing.
