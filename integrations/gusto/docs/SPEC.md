# Slates Specification for Gusto

## Overview

Gusto is a cloud-based payroll, benefits, and HR platform for U.S. businesses. Its API enables partners to manage companies, employees, contractors, payroll processing, benefits administration, tax compliance, and related HR operations. Gusto offers two API programs: App Integrations (for third-party apps connecting with mutual customers) and Embedded Payroll (for building payroll directly into your product).

## Authentication

Gusto uses **OAuth 2.0** exclusively for authentication.

**Credentials required:**

- `client_id` and `client_secret`: Obtained by creating an application in the [Gusto Developer Portal](https://dev.gusto.com).
- `redirect_uri`: Must be registered when creating the application. Wildcards and URI fragments are not supported.

**Authorization flow (Authorization Code Grant):**

1. Direct the user to the authorization URL:
   ```
   https://api.gusto.com/oauth/authorize?client_id={CLIENT_ID}&redirect_uri={REDIRECT_URI}&response_type=code&state={STATE}
   ```
2. The user logs in, selects a company to authorize, and is redirected back with an authorization `code`.
3. Exchange the code for an access token by POSTing to `https://api.gusto.com/oauth/token` with `client_id`, `client_secret`, `redirect_uri`, `code`, and `grant_type=authorization_code`.
4. The response includes an `access_token` and `refresh_token`. The access token is sent as `Authorization: Bearer {access_token}` on all API requests.
5. Refresh tokens by POSTing to the same token endpoint with `grant_type=refresh_token` and the `refresh_token`.

**Token types:**

- **Company-level tokens**: Obtained via the OAuth2 authorization code flow above. Each token is scoped to a single company. If a user manages multiple companies, they must authorize each company individually.
- **System access tokens** (Embedded Payroll only): Generated by POSTing to `/oauth/token` with `grant_type=system_access` using your `client_id` and `client_secret`. Used for partner-level operations like creating companies and managing webhook subscriptions.

**Scopes:**
Scopes follow the pattern `{resource}:{action}` (e.g., `employees:read`, `payrolls:write`). Scopes are assigned by Gusto during the production approval process based on your use case and are enforced in production. Only primary admins or full-access admins on a Gusto account can authorize an application.

**Environments:**

- Demo/Sandbox: `https://api.gusto-demo.com`
- Production: `https://api.gusto.com`

## Features

### Company Management

Create and manage companies, including company profile details, locations, bank accounts, departments, and signatories. For Embedded Payroll, partners can create and onboard "partner-managed companies" programmatically. Companies must be onboarded and approved by Gusto before payroll can be run.

### Employee Management

Create, onboard, update, terminate, and rehire W-2 employees. Supports both employer-driven and employee self-onboarding. Manage employee personal details, home and work addresses (with effective dating), jobs, compensation, payment methods, and tax information. Addresses and compensation support effective dating for scheduled changes.

### Contractor Management

Create, onboard, deactivate, and reactivate contractors (1099 workers). Manage contractor profiles, payment methods, and self-onboarding invitations. Process contractor payments individually or in groups.

### Payroll Processing

Run regular, off-cycle, termination, and historical payrolls. Calculate employee gross wages, submit payrolls for processing, and retrieve payroll results including pay stubs and receipts. Configure pay schedules (weekly, biweekly, semi-monthly, monthly). Payrolls go through multiple statuses (unprocessed → calculated → submitted → processed → paid). Some payroll operations are asynchronous.

### Benefits Administration

Create and manage company-level benefit types (health insurance, 401(k), HSA, etc.) and enroll employees in those benefits. Supports effective dating for benefit changes. Integrates with product partners like SimplyInsured (health insurance), Guideline and Vestwell (401(k)), and NEXT (workers' compensation).

### Tax Management

Configure company and employee federal/state tax information. Gusto handles tax calculations, filings, and form generation (W-2s, 1099s, etc.).

### Forms and Documents

Manage tax forms (W-2, W-4, 1099, I-9), company forms, and contractor documents. Supports form signing, generation, and retrieval. Includes I-9 employment verification workflows.

### Time Off Management

Create and manage time off policies with configurable accrual methods. Create holiday pay policies. Track employee time off balances.

### Reporting

Generate custom reports and retrieve general ledger reports for accounting integration purposes.

### Earning Types

Manage custom earning types for a company (e.g., bonuses, commissions, tips) beyond the standard earning types.

### Garnishments

Create and manage child support garnishments and other wage garnishments for employees.

### Partner Notifications

Receive structured notifications about payroll-critical events (e.g., tax form filings, payroll blockers). Notifications include category, actionability status, and whether the issue blocks payroll.

## Events

Gusto supports **webhooks** for real-time event notifications. Webhook subscriptions are created via API or the Developer Portal and require HTTPS endpoints. Events are signed with an HMAC signature (`X-Gusto-Signature` header) for verification. Failed deliveries are retried up to 16 times over 3 days with exponential backoff.

### Bank Account Events

Notifications when company or employee bank accounts are created, updated, or deleted.

### Company Events

Notifications for company lifecycle changes: provisioned, deprovisioned, partner authorized, migrated, updated, onboarded, approved, needs approval, marked as high-risk/marijuana business, suspended, unsuspended, and payroll transfer completed.

### Company Benefit Events

Notifications when company-level benefits are created, updated, or deleted.

### Contractor Events

Notifications for contractor lifecycle changes: created, updated, onboarded, deactivated, reactivated, deleted, and payment method type updated.

### Contractor Payment Events

Notifications when contractor payments are created or cancelled.

### Contractor Payment Group Events

Notifications when contractor payment groups are created or cancelled.

### Employee Events

Notifications for employee lifecycle changes: created, updated, onboarded, marked for self-onboarding, payment method type updated, terminated, rehired, deleted, and invited to partner portal.

### Employee Address Events

Notifications when employee home addresses or work addresses are created, updated, or deleted.

### Employee Benefit Events

Notifications when employee benefit enrollments are created, updated, or deleted.

### Employee Job Compensation Events

Notifications when employee job compensations are created, updated, or deleted.

### Payroll Events

Notifications for payroll lifecycle changes: created, updated, calculated, submitted, processed, paid, reversed, partially reversed, cancelled, and processing failed.

### Pay Schedule Events

Notifications when pay schedules are created or updated.

### Form Events

Notifications when forms are created, updated, or signed.

### Document Events

Notifications when documents are created, updated, or signed.

### Generated Document Events

Notifications when document generation succeeds or fails.

### Location Events

Notifications when company locations are created or updated.

### Signatory Events

Notifications when signatories are created, updated, deleted, or marked for invite.

### External Payroll Events

Notifications when external (historical) payrolls are created, updated, or deleted.

### Fast ACH Config Events

Notifications when fast ACH configuration is activated or deactivated.

### Notification Events

Notifications when partner notifications are created or resolved. Used to power user-facing messaging about payroll-critical events.
