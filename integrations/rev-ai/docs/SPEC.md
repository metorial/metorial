# Slates Specification for Rev AI

## Overview

Rev AI is a speech-to-text and natural language processing API platform by Rev. It offers transcription along with language identification, sentiment analysis, topic extraction, summarization, and translation. It provides machine-generated transcripts from pre-recorded files or in real-time as audio streams, with support for 57+ languages.

## Authentication

Rev AI uses **Bearer token authentication** via a static access token (API key).

The access token is generated by logging in to Rev AI and navigating to the Access Token page, then clicking "Generate New Access Token." Access tokens should be saved securely, as they are only shown once. A maximum of 2 access tokens are allowed at a time. The access token never expires. Regenerating a token invalidates the previous one.

The token is included in API requests via the `Authorization` HTTP header:

```
Authorization: Bearer <REVAI_ACCESS_TOKEN>
```

There is no OAuth2 flow or scopes. All APIs are accessed using the same access token.

## Features

### Asynchronous Speech-to-Text

Processes pre-recorded audio/video files and returns transcripts. Media files can be submitted via a public URL or by uploading a local file as a multipart/form-data request. Output is available as JSON with timestamps, plain text, SRT, or VTT formats.

- **Language**: Supports 58+ languages.
- **Speaker diarization**: Identifies and labels different speakers in the audio. Premium diarization is also available.
- **Translation**: A `translation_config` parameter can be specified when submitting a job to translate the transcript to another language.
- **Summarization**: Generates summaries of completed transcriptions.
- **Captions**: Returns caption output for a transcription job, supporting SubRip (SRT) and Web Video Text Tracks (VTT) output.
- **Profanity filtering**: Can filter profanity from transcription results.
- **Verbatim mode**: Controls whether output includes filler words and speech disfluencies.
- **Remove atmospherics**: Option to remove atmospherics such as `<laugh>`, `<affirmative>`, etc. from the output.
- **Transcriber selection**: Supports a low-cost transcription option using the Reverb Turbo model via `"transcriber": "low_cost"`.
- **Skip postprocessing**: The `skip_postprocessing` parameter skips inverse text normalization, casing, and punctuation steps for faster results or more control over output.

### Streaming Speech-to-Text (Real-Time)

Enables real-time transcription for streaming audio, with features such as punctuation, capitalization, speaker diarization, custom vocabulary, and profanity filtering. Works with both WebSocket and RTMP streams, with a time limit of 3 hours per stream. Supports 9+ languages for streaming.

### Custom Vocabulary

Improves the accuracy of ASR transcription in both streaming and asynchronous modes. Useful for words or terms not in the average dictionary. Vocabularies are submitted as a list of phrases, where each phrase can be one word or multiple words. Custom vocabularies can be pre-compiled and referenced by ID in transcription jobs.

- Up to 6,000 phrases per job for English, up to 1,000 for other languages.

### Topic Extraction

Identifies important keywords and concepts in transcribed speech, offering a fast and accurate way to retrieve core topics. The API accepts a transcript and returns a ranked list of topics with relevant content fragments.

- Currently only supports English language input.
- Input longer than 14,000 words will fail.

### Sentiment Analysis

Provides additional speech insights by analyzing sentiments in a given transcript. Returns a score in the range [-1, 1], where below -0.3 indicates negative sentiment, above 0.3 indicates positive sentiment, and in between indicates neutral. Results can be filtered by sentiment type (positive, negative, or neutral).

- Currently only supports English language input.
- Input longer than 14,000 words will fail.

### Language Identification

Identifies the spoken language in audio input. Returns confidence scores in the range [0, 1] representing how confident the model is for each detected language.

### Forced Alignment

Provides improved accuracy for per-word timestamps for a transcript. Requires submitting both an audio file and a transcript text file and returns the transcript with precise word-level timing information.

## Events

Rev AI supports webhooks for job completion notifications across all its asynchronous APIs.

### Job Completion

When submitting a job, a webhook URL is supplied in the `notification_config` parameter. Once job processing is complete, the API sends an HTTP POST message to the webhook URL containing the results of processing.

- Applies to all asynchronous job types: Speech-to-Text, Sentiment Analysis, Topic Extraction, Custom Vocabulary, Language Identification, and Forced Alignment jobs.
- Authorization headers can be provided in the `notification_config` parameter for webhook URLs with restricted access.
- The webhook POST body contains job details including job ID, status (success or failure), and failure reason if applicable.
- If a webhook invocation does not receive a 200 response, Rev AI will retry the callback URL every 30 minutes until either 24 hours have passed or a 200 response is received.
